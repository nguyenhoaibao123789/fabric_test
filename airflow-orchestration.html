02
How Airflow Handles This

Dependency Resolution

Airflow uses the >> operator to wire dependencies. Every task uses trigger_rule='all_success' — the default. Whether Silver 2 is a UNION or a JOIN, it only runs when every upstream Silver 1 has succeeded. One failed source stops the chain; fix it and rerun from that point.

🔗 The Dependency Syntax
Three tasks, where silver2 waits for both upstream branches to finish before it can run.
# Define tasks t_bronze_fedex = FabricNotebookOperator(...) t_bronze_dhl = FabricNotebookOperator(...) t_silver1_fedex = FabricNotebookOperator(...) t_silver1_dhl = FabricNotebookOperator(...) t_silver2 = FabricNotebookOperator(...) # Wire dependencies t_bronze_fedex >> t_silver1_fedex t_bronze_dhl >> t_silver1_dhl # silver2 waits for BOTH silver1 tasks [t_silver1_fedex, t_silver1_dhl] >> t_silver2
📋 Scheduling Rules
Airflow's scheduler evaluates these rules on every tick.
✓
Task queued only when all direct upstreams are success
∥
Independent tasks (fedex + dhl) run in parallel automatically — no extra config
✕
Any upstream failure cascades upstream_failed to all downstream tasks — Union or Join, the rule is the same: every input must be clean
↺
Partial rerun: clear one failed task's state → Airflow re-evaluates downstream automatically
⊕
Dynamic tasks: generate 108 task groups from config table at DAG parse time
🔄 Scoped Dependencies — Silver 2 wires only its own sources, all must succeed
Each Silver 2 entity declares only the Silver 1 tasks it feeds from (via silver2_entity in config). All of them must be green. all_success is the rule — union or join, no partial data enters Silver 2.
# Build per-source task groups — collect silver1 tasks by entity silver1_by_entity = {} # { 'carrier_invoice': [task_fedex, task_dhl, ...] } for src in SOURCES: with TaskGroup(f"group_{src['source_name']}") as grp: bronze = FabricNotebookOperator(task_id=f"bronze_{src['source_name']}", ...) silver1 = FabricNotebookOperator(task_id=f"silver1_{src['source_name']}", ...) bronze >> silver1 entity = src['silver2_entity'] silver1_by_entity.setdefault(entity, []).append(silver1) # Silver 2 — all_success (default): every source must be clean before union/join for entity, upstreams in silver1_by_entity.items(): s2 = FabricNotebookOperator(task_id=f"silver2_{entity}", ...) upstreams >> s2 >> t_gold_fact_invoice # no trigger_rule needed — all_success is default
03
Resilience Patterns

Error Handling & Rerun

Four patterns work together so a single source failure never takes down the whole pipeline — and recovery is surgical, not a full restart.

🔁
1 · Auto Retry with Backoff
Transient failures (network blip, API timeout) retry automatically. retries=3 with exponential backoff before marking the task truly failed.
FabricNotebookOperator( task_id="ingest_fedex", retries=3, retry_delay=timedelta(minutes=5), retry_exponential_backoff=True, ) # attempt 1 → fail → wait 5m # attempt 2 → fail → wait 10m # attempt 3 → fail → wait 20m → FAILED
🎯
2 · Partial Rerun — Same run_id
Clearing a task reruns it inside the same DAG run. The run_id never changes — only try_number increments. Already-successful tasks keep their state untouched.
# Same run_id, new try_number # run_id: scheduled__2026-03-23T06:00:00+00:00 ← unchanged # ingest_dhl try=1 → failed # ingest_dhl try=2 → success (after clear) airflow tasks clear medallion_morning_batch \ -t "group_dhl_direct_invoice.bronze_dhl_direct_invoice" \ -d "2026-03-23" # ✓ bronze_fedex + silver1_fedex stay success # ✓ bronze_ups + silver1_ups stay success # ✓ only dhl branch + downstream re-queue
👁️
3 · Cross-DAG Sensors
If Silver 2 is a separate DAG (e.g., runs daily vs. hourly ingest), use ExternalTaskSensor to block Gold until Silver is confirmed complete.
# gold_dag.py — waits for silver DAG success wait_for_silver = ExternalTaskSensor( task_id="wait_silver2_done", external_dag_id="silver_pipeline", external_task_id="silver2_carrier_invoice", timeout=3600, mode="reschedule", # don't block a worker )
🔔
4 · Failure Alerts & SLA Miss
Task-level callbacks fire on failure, retry, or SLA miss. Sends a Teams message via webhook so the on-call person knows exactly which source and which layer failed.
def on_failure(context): source = context["task_instance"].task_id send_teams_alert( title=f"❌ Failed: {source}", dag=context["dag"].dag_id, run_id=context["run_id"], ) default_args = { "on_failure_callback": on_failure, "sla": timedelta(hours=2), }
04
The DAG Definition

One Python File, 108 Sources

The entire orchestration is defined in a single medallion_pipeline.py file. At parse time, Airflow reads sources.json and generates all 108 task groups dynamically — no copy-paste, no per-source files.

fabric/dags/medallion_pipeline.py
""" Fabric Managed Airflow — Medallion Pipeline DAG Metadata-driven: reads sources.json at parse time to generate task groups. """ import json from datetime import datetime, timedelta from pathlib import Path from airflow.decorators import dag from airflow.utils.task_group import TaskGroup from airflow.sensors.external_task import ExternalTaskSensor from airflow_fabric.operators import FabricNotebookOperator # Fabric custom operator from airflow_fabric.operators import FabricSQLOperator from callbacks import on_failure_teams_alert # ── Config ────────────────────────────────────────────────────────── ENV = Variable.get("environment", default_var="dev") SOURCES = json.loads(Path("config/sources.json").read_text()) SOURCES = [s for s in SOURCES if s["is_enabled"] and s["environment"] == ENV] default_args = { "retries": 3, "retry_delay": timedelta(minutes=5), "retry_exponential_backoff": True, "on_failure_callback": on_failure_teams_alert, "sla": timedelta(hours=4), } # ── DAG ───────────────────────────────────────────────────────────── @dag( dag_id="medallion_pipeline", schedule_interval="0 6 * * *", start_date=datetime(2026, 1, 1), default_args=default_args, catchup=False, tags=["medallion", "fabric", ENV], ) def medallion_pipeline(): # ── Phase 1 + 2: Bronze + Silver 1, per source ────────────────── all_silver1_tasks = [] for src in SOURCES: with TaskGroup(f"group_{src['source_name']}") as grp: # Bronze: land raw file in OneLake bronze = FabricNotebookOperator( task_id=f"bronze_{src['source_name']}", notebook_name="bronze/ingest_file", parameters={**src, "layer": "bronze"}, ) # Silver 1: clean + MERGE into staging_{source_name} silver1 = FabricNotebookOperator( task_id=f"silver1_{src['source_name']}", notebook_name=f"silver1/{src['silver1_notebook']}", parameters={**src, "layer": "silver1"}, ) bronze >> silver1 # silver1 waits for bronze all_silver1_tasks.append(silver1) # ── Phase 3: Silver 2 — scoped per entity, all_success (default) ── silver1_by_entity = {} for src, task in zip(SOURCES, all_silver1_tasks): silver1_by_entity.setdefault(src['silver2_entity'], []).append(task) silver2_tasks = [] for entity, upstreams in silver1_by_entity.items(): silver2 = FabricNotebookOperator( task_id=f"silver2_{entity}", notebook_name=f"silver2/build_{entity}", # all_success is default — every upstream must be green (union or join) ) upstreams >> silver2 silver2_tasks.append(silver2) # ── Phase 4: Gold — stored proc in Fabric Data Warehouse ──────── gold = FabricSQLOperator( task_id="gold_fact_invoice", sql="EXEC gold.load_fact_invoice", warehouse="fabric_data_warehouse", ) silver2_tasks >> gold medallion_pipeline()
